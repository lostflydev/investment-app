REST API — методичка

HttpClient, CompletableFuture, Jackson и JavaFX Task — на примере MOEX ISS.

1. Что такое REST

REST (Representational State Transfer) — стиль построения веб-сервисов. Простые правила: ресурсы с URL, HTTP-методы для действий, JSON в телах.

GET    /iss/securities           список всех бумаг
GET    /iss/securities/SBER      одна бумага
POST   /orders                   создать заявку
PUT    /orders/42                обновить заявку
DELETE /orders/42                отменить

2. HTTP в двух словах

Запрос: метод + URL + заголовки + (тело). Ответ: статус-код + заголовки + (тело).

Идемпотентность и безопасность

Запомнить: GET только для чтения.

4. HttpClient из JDK 11+

Синхронный (плохо для UI)

HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create("https://iss.moex.com/iss/securities.json"))
        .GET()
        .build();

HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());

Метод send блокирует вызывающий поток.

Асинхронный (правильно)

CompletableFuture<String> future = client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
        .thenApply(HttpResponse::body);

future.thenAccept(body -> System.out.println(body));

Цепочка CompletableFuture

moexClient.getAsync("/iss/.../securities.json")
        .thenApply(parser::parseBlock)
        .thenApply(this::mergeWithMarketdata)
        .thenAccept(this::renderToTable)
        .exceptionally(ex -> {
            showAlert(ex.getMessage());
            return null;
        });

5. Ошибки и таймауты

Три вещи:
  1. Connect timeout и request timeout — задавайте явно.
  2. Проверяйте статус-код.
  3. Не декодируйте JSON без try-catch.
HttpClient.newBuilder()
        .connectTimeout(Duration.ofSeconds(5))
        .build();

HttpRequest.newBuilder()
        .uri(...)
        .timeout(Duration.ofSeconds(10))
        .build();

if (response.statusCode() / 100 != 2) {
    throw new MyApiException("HTTP " + response.statusCode());
}

6. Парсинг JSON: Jackson

// build.gradle
implementation 'com.fasterxml.jackson.core:jackson-databind:2.17.2'

POJO

public class MoexResponse {
    public Block securities;
    public Block marketdata;
    public static class Block {
        public List<String> columns;
        public List<List<Object>> data;
    }
}

ObjectMapper mapper = new ObjectMapper();
MoexResponse resp = mapper.readValue(body, MoexResponse.class);

JsonNode

JsonNode root = mapper.readTree(body);
JsonNode data = root.path("securities").path("data");
for (JsonNode row : data) {
    System.out.println(row.get(0).asText());
}

7. UI-поток JavaFX и HTTP

JavaFX однопоточный относительно UI. Всё, что касается Scene, Node, TableView, может трогать только JavaFX Application Thread. Если он занят — окно замерзает.

Антипример 
@FXML
public void onLoad() {
    String body = httpClient.send(req, ...).body();
    List<MoexSecurity> data = parse(body);
    table.setItems(FXCollections.observableArrayList(data));
}
Пока идёт запрос (1–10 сек) — окно не реагирует.
Правильно — Task<T> 
@FXML
public void onLoad() {
    Task<List<MoexSecurity>> task = new Task<>() {
        @Override
        protected List<MoexSecurity> call() throws Exception {
            return service.loadTopShares(10).get();
        }
    };
    task.setOnRunning   (e -> statusLabel.setText("Загрузка..."));
    task.setOnSucceeded (e -> table.setItems(FXCollections.observableArrayList(task.getValue())));
    task.setOnFailed    (e -> showAlert(task.getException().getMessage()));

    new Thread(task).start();
}
Коллбэки setOnXxx FX вызывает в UI-потоке.

Platform.runLater

Platform.runLater(() -> statusLabel.setText("Готово"));

8. Чек-лист

  • Задан connect timeout (5–10 сек).
  • Задан request timeout на уровне HttpRequest.
  • Проверен статус-код.
  • Парсинг JSON в try/catch.
  • Запрос запущен в Task + new Thread().start().
  • Обновления UI в setOnSucceeded / setOnFailed или Platform.runLater.
  • Кнопка «Загрузить» блокируется на время запроса.
